git help config 命令不需要联网也可以再浏览器中打开帮助手册 不想要查看完整的手册可以用-h选项获取更简明的 help输出 git config -h git status命令查看文件处于什么状态 精简的写法 是git status -s s代表short的简写 这个时候没有跟踪的前面有红色的? 使用命令 git add开始跟踪一个文件 所以要跟踪 index.html文件 运行以下代码 git add index.html 2.还可以把已跟踪、且已修改的文件放到暂存区 3.把有冲突的文件标记为已解决状态 绿色代表已经到暂存区了 git add . 将里面项目全部进行跟踪 现在暂存区中有一个index.html文件等待被 提交到 git仓库中进行保存 可以执行git commit命令进行提交 其中 -m选项后面是本次的提交消息 用来对提交的内容做进一步的描述 git commit -m "新建了index.html文件" 再次运行 git commit -m "提交信息"命令 就可以把暂存区中记录的index.html的快照提交到Git仓库中进行保存 撤销对文件的修改 把工作区中对应的文件的修改 还原成git仓库中所保存的版本 所有的修改会丢失 且无法恢复 危险性比较高 git checkout -- 文件名 就可以撤销了 如果需要从暂存区中移除相对应的文件 可以使用 gitreset HEAD 要移除的文件名称 .代表所有 如果要跳过 暂存区直接提交到git仓库保存 就需要用到命令 git commit -a -m"描述消息"
